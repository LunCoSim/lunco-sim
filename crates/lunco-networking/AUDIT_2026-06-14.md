## Verdict

**Not production-ready, but it now builds.** The `networking` feature failed to
compile on HEAD (half-finished snapshot-quantization migration — now fixed).
Underneath
that, the wire is **wide open**: an all-zero netcode key, no certificate
validation on native, an authorization gate that only guards two of the five
networked commands, and an unbounded binary decoder. None of this is a design
flaw — the *architecture* (deterministic identity substrate, server-authoritative
ownership, smooth correction, D7 feature-gating) is genuinely well-built and
well-tested. The problems are an unfinished refactor and a security/robustness
layer that was scaffolded for the localhost spike and never hardened. The
prediction/reconciliation core — the thing most likely to make multiplayer feel
bad — was **not audited** (verifier budget ran out); that's the biggest unknown.

## P0 — Blockers

### 1. 🔬 ✅ FIXED — `networking` feature did not compile (half-finished snapshot migration)
`wire.rs:71-90` · `wire.rs:623` · `server.rs:270`

> **Resolved 2026-06-14.** Both encoders now write `pos_q`/`rot_packed` via
> `quantize_pos`/`encode_quat`; change-detection compares the quantized form
> (matching the already-migrated `last_sent`); the missing `bevy::math::DVec3`
> import (which also broke `quantize_pos`/`dequantize_pos` themselves) was added
> to `wire.rs` and `server.rs`; dead `arr3_eq`/`arr4_eq` removed; roundtrip test
> updated. `cargo check --features networking` = clean; `codec_roundtrip` tests pass.
> Original diagnosis below for the record.

Commit `58df3bc4` ("compact snapshot quantization") changed `SnapshotEntry` to
`{ gid, pos_q:[i32;3], rot_packed:u32, lv, av, last_input_seq }` and updated the
**decoder** (`wire.rs:500-518` reads `e.pos_q`, `e.rot_packed`). But **both
encoders** still construct the struct with the deleted fields `t / r / pos / cell`:
- `gather_snapshot` — `wire.rs:623-632`
- `on_server_connected` (connect baseline) — `server.rs:270-286`

→ `error[E0560] struct SnapshotEntry has no field named t` and
`error[E0063] missing fields pos_q, rot_packed`. The roundtrip test
(`wire.rs:723`) uses the old fields too.

**Fix (mechanical):** in both construction sites replace
`t / r / pos / cell` with `pos_q: quantize_pos(<abs DVec3>)` and
`rot_packed: encode_quat(<Quat>)` (helpers already exist at `wire.rs:99` /
`wire.rs:120`); update the roundtrip test; add a CI job that builds
`--features networking` so this can't regress silently.

## P1 — Security / robustness (the wire is open)

All five below were **self-verified from source** after their automated
verifiers were killed.

### 2. 🔬 All-zero netcode private key + hardcoded protocol id → forgeable connect tokens
`shared.rs:13-14` · server binds `0.0.0.0` (`server.rs:133`)

`PRIVATE_KEY: [u8;32] = [0u8;32]` is the netcode shared secret, baked into both
server and client. Netcode's security assumes this key is a server-only secret;
all-zero means anyone can forge a valid connect token offline and join any host.
Combined with the open authz below, that's unauthenticated remote control.
**Fix:** load the key from env/file; fail closed (refuse to start as a
non-loopback host) when it's the placeholder; issue per-client tokens server-side.

### 3. 🔬 `authorize()` guards only Drive/Brake — Spawn/Possess/Release fall through to `Ok(())`
`session.rs:459-469`

```rust
match type_name {
    "DriveRover" | "BrakeRover" => { /* ownership check */ }
    _ => Ok(()),   // SpawnEntity, PossessVessel, ReleaseVessel all pass
}
```
Any client can `SpawnEntity` without limit (DoS), and possession is only as safe
as `SessionRegistry::claim` arbitration. **Fix:** per-command policy — spawn
quota/rate per session, restrict `PossessVessel` to vessel-marked entities,
reject commands against non-target gids; stop treating "not Drive/Brake" as
"unconditionally allowed".

### 4. 🔬 Unbounded binary decode → attacker length-prefix pre-allocates GBs (wasm 4 GiB OOM)
`shared.rs:25-26`

`bincode::deserialize(bytes)` with the default *limitless* config. `WireEnvelope`
holds `Vec<SnapshotEntry>`, `Vec<(u64,u64)>` and `String`s; bincode reads a length
prefix and pre-allocates capacity **before** reading payload, so a crafted frame
declaring a 4-billion-element vec tries to allocate gigabytes — instant OOM on a
wasm client (4 GiB cap = project law). **Fix:** `bincode::DefaultOptions::new()
.with_limit(MAX_FRAME_BYTES).deserialize(...)`, plus post-decode length sanity
caps and a lightyear per-packet size cap.

### 5. 🔬 Native client disables TLS certificate validation for all builds
`client.rs:95-100` · `Cargo.toml:52` (`webtransport_dangerous_configuration`)

Native `client_cert_digest()` returns `String::new()` and the dangerous-config
feature is compiled unconditionally → the native client accepts **any** cert from
**any** server at the target address (MITM). Framed as "localhost dev" but not
gated to loopback. **Fix:** gate dangerous-config/empty-digest to an explicit dev
flag or loopback-only; for non-loopback `--connect` require a pinned digest or
real CA validation.

### 6. 🔬 Wire command apply lacks the constructibility guard the HTTP path has → malformed packet can panic
`wire.rs:448-461` vs `executor.rs:145-161`

The HTTP dispatcher checks `ReflectFromReflect::from_reflect(...).is_some()`
*before* `reflect_event.trigger(...)` (executor.rs:147-148), with a comment that
`trigger` falls back to `Default`/`FromWorld` and **panics** when neither applies.
`apply_wire_command` (wire.rs:460) calls `trigger` directly with no such guard —
a malformed/partial wire command panics the app (violates app-must-never-stall).
**Fix:** mirror the dispatcher's guard; warn+drop on non-constructible.

### Remaining security findings (⚠️ code-cited, auto-verifier killed)

- **⚠️ No per-connection rate limit / quota** (`server.rs:350-362`) —
  `host_recv_inbox` pushes every inbound envelope into `WireInbox` unbounded; a
  client can flood the reliable `CmdChannel` or spam `SpawnEntity` at line rate.
  Fix: per-session token bucket, bound `WireInbox`, entity-spawn quota.
- **⚠️ Replay/dedupe key is a client-minted OpId from a predictable LCG**
  (`ids.rs:55-59`, seed `12345`) — `WireDedup` rejects repeats but the host trusts
  the inbound `m.id` (`wire.rs:496`); OpIds are predictable, so one client can
  poison another's dedup space. Fix: namespace the dedup key per trusted origin
  session; mint OpIds from a CSPRNG (or a host-validated per-session sequence).
- **⚠️ Client fully trusts host snapshots/spawns — no caps** (`wire.rs:499-527`) —
  no bound on entries-per-snapshot, total replicated bodies, or
  `PendingReplicatedSpawns`; `last_sent` (`wire.rs:564`) never pruned. A hostile or
  buggy host (relevant given #5) can OOM the client. Fix: cap queue lengths +
  total instantiated bodies; prune `last_sent`.
- **⚠️ `PeerId::Raw(_) => SessionId(u64::MAX)`** (`shared.rs:31-38`) — all raw
  peers collapse to one session id, so two such peers would share ownership in
  `SessionRegistry`. Fix: map by inner `n`, or reject peers resolving to the
  sentinel.

## P1/P2 — Identity & determinism substrate (always-on)

All auto-verified. The substrate is solid; these are edges.

- **✅ medium — Known instancing-collision gap has no debug-time guard**
  (`lib.rs:283-289`). The same logical asset instanced twice can derive an
  identical `GlobalEntityId`; `SkipContentStamp` mitigates *runtime* spawns, but
  `assign_global_entity_ids` never checks derived-id uniqueness even though the
  design promises a "loud, not silent" check. Fix: `cfg(debug_assertions)`
  collision guard (Local map of id→entity) that `warn!`/`debug_assert!`s on a
  duplicate, naming both entities + provenance.
- **✅ medium — Untagged-entity fallback mints a non-deterministic id, unsynced**
  (`lib.rs:524-539`). The `None` arm calls `allocate_authoritative()` with **no
  `is_server` gate** (the Authoritative + runtime arms *are* gated). An
  un-migrated spawner that should replicate gets different ids on host vs client
  → permanent desync, warned only once. Fix: gate the `None` arm on `is_server.0`;
  aggregate the warning instead of once-ever.
- **✅ low — `make_id_53` sequence overflow carries into the timestamp field**
  (`ids.rs:38-42`). Within one second the increment masks to 53 bits, not the
  21-bit seq sub-field; >2²¹ ids/sec corrupt the time-sorted invariant. Fix:
  detect seq exhaustion explicitly, bump timestamp deliberately.
- **✅/⚠️ low — `IsServer` defaults to `true`** (`lib.rs:370-377`). A client that
  spawns before the net plugin flips it to `false` mints ids the host won't agree
  on (fail-open). Fix: set `IsServer(false)` earliest in the client plugin; log
  the resolved value; debug-check Client-role ⇒ `IsServer(false)`.
- **✅ low — `from_raw` is a public mint-by-the-back-door** (`lib.rs:129-135`).
  Contract is doc-only. Optional: `reconstruct_from_wire` name / debug-assert.
- **❌ (downgraded) — Content id separator ambiguity** (`identity.rs:92-98`).
  Abstractly true (`:`-joined, no length-prefix) but the verifier showed every
  concrete reachable collision is false: `source` is a registered scheme name, not
  a raw `twin://…` string. Nice-to-have hardening (length-prefix fields), not a bug.
- **❌ — Derived id entropy reduction** (`identity.rs:100-106`): verifier found no
  action needed for MVP.
- **(non-issue) — SimTick advance under TimeWarp** (`lib.rs:469-474`): verified
  *correct* and warp-independent — effectively a strength.

## P2 — Integration seam (`sandbox.rs`)

- **✅ low — `networked` keepalive flag diverges from the actual mount decision**
  (`sandbox.rs:118`). Window pacing scans `argv` (`--host`/`--connect`); the mount
  uses `NetworkMode::resolve()` which on wasm reads `?connect=` from the URL and
  ignores argv. A wasm `sandbox` build (a supported networked target) connects but
  doesn't get the keepalive present-mode. Fix: compute `NetworkMode::resolve()`
  once; derive `networked = mode.is_some()`; reuse at the mount site.
- **✅ low — order doc / dual SimTick writer** (`sandbox.rs:415`, `wire.rs:519`):
  client `drain_wire_inbox` (Update) hard-sets `SimTick` to the host tick while
  `advance_sim_tick` (FixedUpdate) increments it. Deliberate but undocumented;
  add a comment + a convergence test. Correct as-is.

## Strengths (genuinely well-built)

- **Deterministic identity is rigorous.** `identity.rs` and the proto-tests
  reference impl are byte-for-byte identical (fnv1a64/fold_53/canonicalize_path/
  derive_id), with a determinism test suite. Minting is funnelled through one
  system (`assign_global_entity_ids`); `GlobalEntityId`'s field is private, no
  `new()`/`Default`. `SimTick` correctly decouples netcode time from time-warp.
- **Always-on substrate is centralized + regression-tested.**
  `register_core_resources` inits every always-on resource with an anti-regression
  test; `SessionRegistry` authority paths avoid nondeterministic iteration order.
- **Server re-attributes command origin from the trusted connection**, never the
  client's claimed origin (`wire.rs:487`) — the right call.
- **Drive/Brake are correctly ownership-gated** with schema-driven authz targets.
- **Malformed frames are dropped, not panicked** at the codec/unknown-type layer
  (`deserialize_env` → `Option`). (The panic risk is specifically the Reflect
  `trigger` path, finding #6.)
- **Dedupe + ownership state are explicitly bounded** (`WireDedup` FIFO 8192,
  `CommandResults` FIFO 1024).
- **Diagnostics are zero-overhead off** (airtight feature gating), read-only (no
  Transform writes / no sim perturbation), panic-free, with correct schedule
  placement (`Last`).
- **The integration seam's no-feature path is a true no-op**; the `AppliedInputSeq`
  trap is actively defended; tick-rate is single-sourced so lightyear `Tick` and
  `SimTick` can't drift.
