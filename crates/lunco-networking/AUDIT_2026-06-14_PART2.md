Completes the 2026-06-14 audit. Part 1 (`AUDIT_2026-06-14.md`) covered 4/12

## Headline

The architecture holds up under the deeper read. The prediction/reconciliation
core — Part 1's "biggest unknown" — is **sound**: all three design invariants
(never-write-Transform / predict-only-owned-driving / sample-after-writeback) are
upheld in the reconcile *logic*. **But the one place that invariant breaks is a
scheduling bug in the host's snapshot sampler** (NEW-1 below) — a silent no-op
`.after(Writeback)` — which means every snapshot the host ships is sampled at the
wrong point in the tick. Highest-value new finding; one-line fix.

Beyond that: the 5 Part-1 P1 security items are all **confirmed and now pinpointed
to exact lines**, and the command-sync layer has a **gap vs. its own design docs**
— applied commands are never re-broadcast to other clients (only ownership/pose/
spawn converge), so the docs' "apply → broadcast → Ack" is unbuilt.

## NEW findings (this pass)

### NEW-1 · 🔬 P1 (high value) — `gather_snapshot` `.after(Writeback)` is a silent no-op (wrong schedule)
`sync.rs:705`

```rust
.add_systems(FixedUpdate, gather_snapshot.after(PhysicsSystems::Writeback));
```
avian's `PhysicsSystems::Writeback` set runs in **`FixedPostUpdate`**, not
`FixedUpdate`. A Bevy `.after(set)` against a set with **zero members in the
target schedule imposes no constraint** — so `gather_snapshot` runs at an
arbitrary point in `FixedUpdate`, i.e. *before* this tick's physics
integration/writeback. The author's own comment (sync.rs:699-704) states the
intent — "sample the pose AFTER avian has integrated this tick … so the
snapshot's pose and its `last_input_seq` ack are a consistent post-step pair —
matching how the client records its predicted pose" — and the sibling client
reconcile/record chain proves the correct pattern: `commands.rs:2471-2481` use
`FixedPostUpdate` + `.after(PhysicsSystems::Writeback)`. The host samples
last-tick's pose stamped with this-tick's ack: the exact 1-tick mispair the
client then has to reconcile away on **every** snapshot.

**Fix (one line):** move it to `FixedPostUpdate`:
```rust
.add_systems(FixedPostUpdate, gather_snapshot.after(PhysicsSystems::Writeback));
```
The 20 Hz steady-clock decoupling (the reason it left `Update`) is preserved —
`FixedPostUpdate` is on the same fixed clock.

### Codec (binary wire) — ✅ mostly sound
- 🔬 **P1 · `shared.rs:21-27` unbounded bincode decode.** `bincode::deserialize`
  (v1.3 free fn, default config) applies **no length limit**; a hostile/corrupt
  frame whose `Vec<SnapshotEntry>`/`String` length prefix is huge pre-allocates
  that many bytes → OOM before any field is read. This is Part-1 P1 #4, now
  pinpointed. **Fix:** `bincode::options().with_limit(MAX).with_fixint_encoding()
  .deserialize(bytes)`, or reject oversize frames pre-decode. (Also `.ok()` at
  shared.rs:21-27 **silently swallows** the decode error — add a `warn!`.)
- ✅ **P2 · `sync.rs:100-106` NaN quantize → origin teleport.** `(NaN * POS_SCALE)
  .round() as i32` yields `0` (Rust `as` maps NaN→0), so a NaN position component
  silently teleports the body to the origin on that axis. Inf saturates correctly
  to ±i32::MAX (≈±2147 km, the documented clamp). **Fix:** skip non-finite
  entries in `gather_snapshot`.
- ✅ **P2 · `sync.rs:~203` no envelope version tag.** `SyncEnvelope` variants are
  bincode positional discriminants; reorder/add a variant across mismatched
  builds → silent desync (decode `Ok(garbage)` or swallowed `Err`). Acceptable
  under "always build together," brittle across rolling deploys. **Fix:** prepend
  a 1-byte format version; never reorder variants.
- ✅ **OK:** `quantize_pos`/`dequantize_pos` round-trip ±0.5 mm, saturating cast,
  no wrap/UB. `encode_quat`/`decode_quat` smallest-three correct (sign-fold,
  clamp, `.max(0.0)` guards sqrt; max err ≈1.4e-3). lv/av lossless. `last_input_seq`
  u32 wrap ≈2.3 yr (non-issue). Endianness consistent (same binary both peers).

### Prediction / reconciliation / smoothing — ✅ invariants hold; tuning nits
- ✅ **Invariants 1/2/3 all UPHELD.** (1) Reconcile parks deltas in
  `PendingCorrection`, drained to `Position`/`Rotation` in `FixedUpdate`
  (commands.rs:1001-1014, 1240-1253, 1329-1359); `Snap` writes Transform by
  design (intended teleport). (2) Predict gated by `predicts_locally(...)`
  (commands.rs:738, 833-835); `ArticulatedVehicle` excluded (commands.rs:1149),
  owned rovers excluded from remote-predict (1168). (3) Reconcile/record chain +
  gather are all `.after(Writeback)` *in intent* — but see **NEW-1**: the host's
  gather lands in the wrong schedule.
- ⚠️ **P1 · `commands.rs:537-553` `interpolate_proxies` writes Transform every
  render frame** for non-RigidBody proxies — resets `bevy_transform_interpolation`
  easing (the invariant-1 hazard). OK *only if* these entities are excluded from
  `interpolate_all()`; otherwise it's a real jitter source for visual-only movers.
  **Action:** confirm the exclusion.
- ⚠️ **P1 · `commands.rs:1038-1047` `Snap` half-blends velocity.** On a gross
  teleport the pose is hard-seated but velocity only half-seated → body drifts off
  the freshly-snapped pose for a tick. **Fix:** fully seat velocity on `Snap`;
  keep half-blend for `Correct` only.
- ⚠️ **P2 · `sync.rs:520,531` `SimTick` can rewind on reordered snapshots.**
  `tick.0 = snap.tick` assigned unconditionally in receive order; a late lower-tick
  snapshot rewinds `SimTick`, which feeds prediction-grace math. **Fix:**
  `tick.0 = tick.0.max(snap.tick)`.
- ⚠️ **P2 · `commands.rs:947-952` `last_reconciled` advances even when history for
  that seq is missing** (pruned past `MAX_PREDICTED_HISTORY=128`) → that divergence
  is never corrected until a higher ack. Mostly self-healing.

### Server authority / robustness
- 🔬 **P1 · `shared.rs:14` all-zero `PRIVATE_KEY`** + 🔬 **P1 · `server.rs:134`
  binds `0.0.0.0`.** (Part-1 P1 #2, confirmed.)
- 🔬 **P1 · `server.rs:349-361` `host_recv_inbox` — no rate limit / per-client
  quota.** Every client frame is pushed to `SyncInbox` unbounded, each driving an
  unbounded bincode deserialize → inbound flood DoS. **Fix:** per-session inbound
  frame budget per tick; drop/disconnect on overflow.
- ⚠️ **P1 · `server.rs:333-345` `host_send_outbox` outbox unbounded.** When the
  `Update`-throttled ferry (≈5 Hz unfocused) is outpaced by the 20 Hz
  `gather_snapshot` producer, `SyncOutbox` grows without bound. **Fix:** cap/coalesce
  (keep latest per gid) before push.
- ⚠️ **P2 · `server.rs:269-300` connect baseline unbounded** (all `NetReplicate`
  into one frame, no cap) · **P2 · `shared.rs:35` `peer_to_session` maps every
  `PeerId::Raw(_)`→`u64::MAX`** (distinct peers collide to one SessionId →
  authority confusion) · **P2 · `server.rs:231,239` `Single<&Server>` panics** on
  0/2 servers in a client-triggered observer · **P2 · `server.rs:81`
  `announce_digest` indexes `chain[0]`** unguarded · **P2 · `server.rs:180-190`
  ownership broadcast** fires on any registry change → claim/release spam =
  full-table reliable broadcast amplification.
- ✅ **OK:** broadcast serializes **once** (`NetworkTarget::All`), not per-client;
  no unwrap on client payload (bad frames dropped); disconnect releases session
  ownership (`server.rs:323`); tick from host `SimTick`, not client-controlled.

### Client connection / protocol
- 🔬 **P1 (security) · `client.rs:84-88` + `Cargo.toml:54` native cert
  verification OFF.** `webtransport_dangerous_configuration` is enabled and
  `client_cert_digest()` returns `String::new()` (client.rs:201-204) → native
  accepts **any** cert (MITM), not just localhost. Part-1 P1 #5, pinpointed.
  **Fix:** gate the dangerous feature + empty digest behind a `dev`/localhost-only
  cargo feature; require a real digest or CA validation for non-loopback addrs.
- ⚠️ **P1 · `client.rs:70` `.expect("netcode client")`** crashes the whole app on
  a bad `JoinServer` instead of failing the connect gracefully.
- ⚠️ **P2 · `client.rs:188-195` `resolve_socket_addr`** silently falls back to
  `127.0.0.1:5888` on DNS failure (typo'd host → connects to localhost) **and**
  blocks the frame on synchronous DNS inside a Bevy system. **P2 · `client.rs:97`**
  no connect timeout/backoff/retry.
- ✅ **OK:** channels symmetric and sane — `CmdChannel`=OrderedReliable
  (commands/handshake/spawn), `SnapChannel`=UnorderedUnreliable (snapshots), both
  Bidirectional, server sends on the same (`server.rs:217-218`), client routes
  match (`client.rs:244-247`). Disconnect resets `LocalSession`→LOCAL +
  NetStatus (`client.rs:188-197`). wasm/native cfg coherent.

  `optional=true` both targets, pulled only via `networking=[...]`, `default=[]`.

### API command codec / id translation
- ⚠️ **P1 (security) · `executor.rs:138-162` vs `sync.rs:449-461` divergent
  reflect-trigger hardening.** The API dispatcher guards `ReflectEvent::trigger`
  with a `ReflectFromReflect` constructibility check (executor.rs:146); the **wire
  apply path does NOT** (sync.rs:461 triggers directly) → a wire command that
  deserializes-but-isn't-FromReflect-constructible **panics the host**. This is
  Part-1 P1 "reflect-trigger no guard," now located precisely. **Fix:** port the
  constructible guard into `apply_sync_command`.
- ⚠️ **P2 (security) · `sync.rs:411` + `executor.rs:213` `authz_target_gid` only
  scans top-level struct fields** (returns `None` for tuple-structs/enums/nested),
  and `authorize`'s `_ => Ok(())` allows by default → any non-Drive/Brake command
  with a nested/enum target authorizes as target-less and bypasses ownership.
  Extends Part-1 P1 #3. **Fix:** recurse in `authz_target_gid`; flip `authorize`
  to deny-by-default for state-producing commands.
- ⚠️ **P2 (security) · `executor.rs:100` `type_name` not allowlisted** — apply
  resolves against the **entire** `AppTypeRegistry`, so a peer can trigger any
  `Event+Reflect+ReflectEvent` type, not just the 5 declared channels. **Fix:**
  reject inbound `type_name` absent from `SyncChannelRegistry` (or whose channel
  is `Local`).
- ⚠️ **P2 · `sync.rs:447` resolve-miss → raw u64 deserialized as `Entity`** (bogus
  live entity instead of clean reject; HTTP path uses a `from_raw(0)` sentinel,
  apply path has none). **Fix:** substitute `Entity::PLACEHOLDER` on resolve miss.
- ⚠️ **P2 · `sync.rs:380` capture ships `"null"` on serialize failure** → silently
  drops the command's params (two disconnected log lines). **Fix:** `warn!` + skip
  the push.

### Gap analysis (docs vs built)
- ⚠️ **P1 gap · server command re-broadcast NOT built.** `PH2_OP_LOG` P2.4 +
  `MVP_MULTIPLAYER_GAPS` claim "apply → broadcast to other clients → Ack origin."
  Built: host applies inbound commands locally only; **no `SyncEnvelope::Command`
  re-broadcast** (only `broadcast_ownership`/`broadcast_new_spawns`/`gather_snapshot`
  emit). Possession converges to peers via `OwnershipMsg`; pose via snapshots;
  spawns via NetSpawn. **Any command whose effect isn't one of those (e.g. a
  parameter/structural edit) never reaches other clients.** **Fix:** re-enqueue
  applied authoritative commands to `SyncOutbox` for `AllExcept(origin)`, OR
  update docs to "ownership+pose+spawn only; structural edits out of MVP scope."
- ⚠️ **P2 gap · `Ack`/`Reject`/`parent_gen` unbuilt.** `SyncEnvelope::Ack` is
  explicitly unused (sync.rs:541 "MVP optimistic"); `Reject::NotAuthorized` only
  `warn!`s (sync.rs:413-419); `Mutation.parent_gen` / `Reject::StaleParent`
  optimistic-concurrency never validated.
- ⚠️ **P2 doc-drift** · `PH2_OP_LOG` P2.2 still shows `data: serde_json::Value`
  (code ships JSON **text** `String`, per the bincode `deserialize_any`
  constraint) and references the old `resolve_ids_in_json` name-heuristic
  (replaced by schema-driven `convert_node`). `declare_replication::<C>` shipped
  as `declare_channel::<C>` (DeclareChannelExt) — built and correct, name drifted.
- ⚠️ **P2 gap · Tier-2 loopback tests absent.** Only `codec_roundtrip` (sync.rs:709)
  + `id_codec_tests` (executor.rs:740) exist; none of the planned 8 headless
  loopback tests (command-arrives / broadcasts / dedupe / no-echo / stale-parent /
  two-clients-spawn / possession-isolation) are implemented.
